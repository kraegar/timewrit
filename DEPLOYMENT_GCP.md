# TimeWrit: The GCP Admin's Playbook (V3) 🏗️🔨🛡️

This guide provides the exact `gcloud` commands required to deploy TimeWrit to Google Cloud Platform. This version includes critical prerequisites for a successful first-time deployment.

---

## 🏗️ Phase 0: Prerequisites (Don't Skip!)

Before running any CLI commands, ensure the following are configured in your GCP Console:

1.  **Billing**: Ensure your GCP Project is linked to an active **Billing Account**.
2.  **OAuth Consent Screen**:
    *   Go to **APIs & Services > OAuth consent screen**.
    *   Select **Internal** (if using Google Workspace) or **External**.
    *   Fill in the App Name, User Support Email, and Developer Contact Info.
    *   *Without this, you cannot create OAuth Client IDs in Phase 6.*

---

## 🚩 Phase 1: Project Initialization

Replace `[PROJECT_ID]` with your unique GCP project ID.

```bash
# 1. Create Project
gcloud projects create [PROJECT_ID]

# 2. Set Active Project
gcloud config set project [PROJECT_ID]

# 3. Enable Required Service APIs
gcloud services enable \
    compute.googleapis.com \
    run.googleapis.com \
    sqladmin.googleapis.com \
    secretmanager.googleapis.com \
    cloudbuild.googleapis.com \
    artifactregistry.googleapis.com \
    storage.googleapis.com

# 4. Create Service Account (The Deployer)
gcloud iam service-accounts create timewrit-deployer \
    --display-name="TimeWrit Deployment Account"
```

## 🔒 Phase 2: Security & Secrets

### Generate Your Keys
```bash
# Generate a secure Django SECRET_KEY
openssl rand -base64 32
```

### Create the Secret Containers
```bash
# Core Secrets
gcloud secrets create DATABASE_URL --replication-policy="automatic"
gcloud secrets create SECRET_KEY --replication-policy="automatic"
gcloud secrets create ALLOWED_HOSTS --replication-policy="automatic"

# Storage & Auth Secrets
gcloud secrets create GS_BUCKET_NAME --replication-policy="automatic"
gcloud secrets create GOOGLE_CLIENT_ID --replication-policy="automatic"
gcloud secrets create GOOGLE_CLIENT_SECRET --replication-policy="automatic"
```

### Add Secret Values
Use the `printf` pattern to avoid terminal character mangling:
```bash
# Example: Adding your Database URL
printf "postgres://django:[PASSWORD]@/timewrit_prod?host=/cloudsql/[PROJECT_ID]:[REGION]:[INSTANCE]" | \
    gcloud secrets versions add DATABASE_URL --data-file=-

# Repeat for all others. Use the value generated by openssl for SECRET_KEY.
```

### Grant Permissions to Service Account
```bash
# Grant Secret Accessor role to your deployer account
gcloud projects add-iam-policy-binding [PROJECT_ID] \
    --member="serviceAccount:timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com" \
    --role="roles/secretmanager.secretAccessor"
```

## 🗄️ Phase 3: Data & Storage (SQL + GCS)

### 1. Cloud SQL Setup
```bash
# Create Postgres Instance
gcloud sql instances create timewrit-db \
    --database-version=POSTGRES_15 --tier=db-f1-micro --region=us-east1

# Create Database & Set Password
gcloud sql databases create timewrit_prod --instance=timewrit-db
gcloud sql users set-password postgres --instance=timewrit-db --password=[DB_PASSWORD]

# Grant SQL Client Role
gcloud projects add-iam-policy-binding [PROJECT_ID] \
    --member="serviceAccount:timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com" \
    --role="roles/cloudsql.client"
```

### 2. GCS Storage Setup
```bash
# Create the Storage Bucket (Must be globally unique)
gsutil mb gs://[YOUR_UNIQUE_BUCKET_NAME]

# Grant Storage Admin (To manage media uploads/deletes)
gcloud projects add-iam-policy-binding [PROJECT_ID] \
    --member="serviceAccount:timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com" \
    --role="roles/storage.objectAdmin"
```

## 📦 Phase 4: Build & Service Deployment

```bash
# 1. Create Artifact Registry
gcloud artifacts repositories create timewrit-repo \
    --repository-format=docker --location=us-east1

# 2. Build and Push Image (Run from local project root)
gcloud builds submit --tag us-east1-docker.pkg.dev/[PROJECT_ID]/timewrit-repo/timewrit-app .

# 3. Deploy to Cloud Run
gcloud run deploy timewrit-app \
    --image us-east1-docker.pkg.dev/[PROJECT_ID]/timewrit-repo/timewrit-app \
    --region us-east1 \
    --service-account timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com \
    --set-env-vars="USE_SECRET_MANAGER=True,GCP_PROJECT_ID=[PROJECT_ID],USE_GOOGLE_OAUTH=True,USE_GCS=True" \
    --add-cloudsql-instances [PROJECT_ID]:us-east1:timewrit-db \
    --allow-unauthenticated
```

## 🛠️ Phase 5: The Bootstrap (Cloud Run Jobs)

### Database Migration Job
```bash
gcloud run jobs create timewrit-migrate \
    --image us-east1-docker.pkg.dev/[PROJECT_ID]/timewrit-repo/timewrit-app \
    --region us-east1 \
    --service-account timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com \
    --command=python \
    --args=manage.py,migrate \
    --set-env-vars="USE_SECRET_MANAGER=True,GCP_PROJECT_ID=[PROJECT_ID],USE_GOOGLE_OAUTH=True" \
    --add-cloudsql-instances [PROJECT_ID]:us-east1:timewrit-db

gcloud run jobs execute timewrit-migrate --region us-east1
```

### Initial Superuser Job
```bash
gcloud run jobs create timewrit-create-admin \
    --image us-east1-docker.pkg.dev/[PROJECT_ID]/timewrit-repo/timewrit-app \
    --region us-east1 \
    --service-account timewrit-deployer@[PROJECT_ID].iam.gserviceaccount.com \
    --command=python \
    --args=manage.py,shell,-c,"from django.contrib.auth import get_user_model; User=get_user_model(); User.objects.filter(username='admin').exists() or User.objects.create_superuser('admin', 'admin@your-domain.com', '[PASSWORD]')" \
    --set-env-vars="USE_SECRET_MANAGER=True,GCP_PROJECT_ID=[PROJECT_ID]" \
    --add-cloudsql-instances [PROJECT_ID]:us-east1:timewrit-db

gcloud run jobs execute timewrit-create-admin --region us-east1
```

## 🌐 Phase 6: Google OAuth Handshake

### 1. Google Console Configuration
Navigate to **APIs & Services > Credentials** and create an **OAuth 2.0 Client ID**.
*   **Authorized JavaScript Origins**: `https://your-domain.com`
*   **Authorized Redirect URIs**: `https://your-domain.com/auth/complete/google-oauth2/`

### 2. Production Hardening (Handshake Secrets)
*   **Redirect URI**: The trailing slash after `google-oauth2/` is MANDATORY.
*   **Proxy Headers**: `USE_X_FORWARDED_HOST = True` is required in the environment to correctly generate secure callbacks.

## 🎓 Phase 7: Onboarding the Team

1.  Ask a team member to log in via Google once (this creates their account).
2.  Log in with your **admin** (Janitor) account created in Phase 5.
3.  Navigate to **Users > [Team Member]** and grant **Staff status** and the **Researchers** group.

> [!NOTE]
> The **Researchers** group is automatically created and maintained via the self-healing migration `0043_create_researchers_group.py`.
